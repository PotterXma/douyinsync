# Product Brief: DouyinSync

## 执行摘要

**DouyinSync** 是一款 Windows 系统托盘应用程序，它能自动监控指定的抖音账号，定时下载新发布的视频及封面，并自动上传到用户自己的 YouTube 频道。

今天，想要将抖音内容同步到 YouTube 的创作者面临着极其繁琐的手动工作流程——手动下载视频、转换封面格式、填写 YouTube 元数据、逐一上传。每天花费 30-60 分钟在这些重复性劳动上。DouyinSync 将这个过程完全自动化，用户只需要配置一次抖音账号和 YouTube 凭据，之后一切自动完成。

这个工具利用从开源项目 TikTokDownloader 提取的成熟抖音下载技术，结合 YouTube Data API v3，构建一个可靠的、常驻后台的、可灵活调度的同步管道。

## 核心问题

跨平台内容同步是一个高度重复、低价值但不可省略的日常任务：

- **手动下载**：需要逐一复制抖音链接，使用下载工具获取视频文件
- **封面处理**：抖音封面格式(WebP)不被 YouTube 直接支持，需要手动转换
- **元数据迁移**：标题、描述、标签需要手动复制粘贴
- **上传等待**：YouTube 上传速度受限，一个视频可能需要数分钟
- **状态追踪**：哪些视频已同步、哪些还没有，全靠人脑记忆

这些步骤单独看都不复杂，但每天重复执行极其消耗时间和精力，并且容易出错（遗漏视频、重复上传）。

## 解决方案

一个**常驻 Windows 系统托盘的后台应用**，核心能力：

1. **灵活的定时调度** — 支持两种模式：
   - 固定间隔（每 N 小时执行一次）
   - 周期定时（每天/每周的指定时间执行）

2. **自动抖音抓取** — 用户配置目标抖音账号后，自动：
   - 获取账号最新发布作品列表
   - 提取最高画质视频下载地址
   - 下载视频文件和静态封面

3. **自动 YouTube 上传** — 使用原始抖音文字（标题、描述、标签）直接上传：
   - 视频文件上传
   - 封面图片设置为 YouTube 缩略图（自动格式转换）
   - 元数据填充（标题、描述、标签）

4. **智能去重** — SQLite 数据库记录已同步的作品 ID，避免重复处理

5. **系统托盘交互** — Windows 右下角图标，右键菜单提供：
   - 查看同步状态
   - 手动触发同步
   - 修改配置
   - 查看日志

## 独特价值

- **一站式解决**：市面上的工具大多只解决下载或上传中的一环，DouyinSync 打通整条链路
- **常驻后台**：不需要每天手动启动，开机即运行
- **成熟的下载引擎**：基于 TikTokDownloader 经过社区验证的抖音数据采集能力，包含完整的反爬解决方案
- **灵活调度**：支持多种定时策略，适应不同使用习惯

## 目标用户

**主要用户**：拥有自己 YouTube 频道的内容搬运者或多平台创作者，需要将抖音上的优质内容定期同步到 YouTube，扩大内容覆盖范围。

- 个人创作者两端同步自己的内容
- 内容策划者运营多抖音账号并汇集到 YouTube
- 技术能力中等，能够配置 Cookie 和 API 凭据

## 成功标准

| 指标 | 目标 |
|------|------|
| 同步成功率 | ≥ 95% 的新视频在 24h 内完成同步 |
| 人工干预频率 | 每周 ≤ 1 次（仅 Cookie 过期等异常） |
| 启动到首次同步 | 配置完成后 15 分钟内启动首次自动同步 |
| 系统资源占用 | 空闲时 CPU < 1%, 内存 < 50MB |

## MVP 范围

### ✅ 首版包含

- 可配置的抖音账号列表（发布作品）
- 视频 + 封面下载
- YouTube 视频上传 + 缩略图设置
- 使用原始抖音文字作为 YouTube 元数据
- 固定间隔 + 周期定时两种调度模式
- Windows 系统托盘应用，右键菜单操作
- SQLite 去重数据库
- 本地日志记录

### ❌ 首版不含

- GUI 设置界面（首版使用配置文件）
- 多语言翻译
- 抖音直播录制
- YouTube 播放列表管理
- 多 YouTube 频道支持
- Mac/Linux 支持

## 愿景

如果 DouyinSync 验证成功，未来可以发展为：

- **可视化管理面板**：Web UI 管理所有同步任务
- **双向同步**：YouTube → 抖音方向
- **多平台扩展**：支持 TikTok、B站、小红书等
- **智能元数据**：AI 辅助翻译和优化视频标题/描述
- **批量历史同步**：一键同步某账号的全部历史视频

## 技术方案概要

| 维度 | 选型 |
|------|------|
| 语言 | Python 3.10+ |
| 抖音采集 | 基于 TikTokDownloader 提取的核心模块 |
| YouTube 上传 | google-api-python-client (YouTube Data API v3) |
| HTTP 客户端 | httpx (async) |
| 定时调度 | APScheduler |
| 系统托盘 | pystray + Pillow |
| 数据库 | SQLite (aiosqlite) |
| 封面处理 | Pillow (WebP → JPEG 转换) |
| 打包分发 | PyInstaller |

### YouTube API 配额注意事项

- 默认每日 10,000 单位配额
- 视频上传消耗 1,600 单位
- 缩略图设置消耗 50 单位
- 每日理论上限约 6 个视频上传（含缩略图）
- 如需更多，可申请配额提升
